# Assets

This folder holds visual and media assets for the event.

## Folder Structure

```
assets/
  images/       ← Logos, banners, screenshots
  diagrams/     ← Flow diagrams, architecture visuals
  newsletters/  ← Archived newsletter or announcement files
```

## Usage Notes

- Images used in docs should be placed in `assets/images/` and referenced with relative paths
- Diagrams explaining the event format or qualification flow belong in `assets/diagrams/`
- Keep file names lowercase and hyphenated (e.g., `event-banner.png`, `qualification-flow.svg`)
- Prefer SVG over PNG for diagrams where possible — they scale cleanly and are version-control friendly

## What Belongs Here

- Community or event logos
- Event banners for social sharing
- Flow diagrams (registration flow, qualification flow, feedback flow)
- Slide decks or presentation files (exported formats)
- Archived newsletter content

## What Does Not Belong Here

- Large binary files (videos, high-res assets over a few MB)
- Files that change frequently — link to external storage instead
