# Overview

## What {{event_name}} Is

{{event_name}} is a {{event_duration_weeks}}-week, participation-driven builder sprint organized by {{community_name}}.

The purpose is to create a lightweight structure that encourages people to:

- build consistently
- share progress publicly
- ship real products
- test ideas with real users
- learn in public with clear expectations

This event is not designed around judges ranking winners. It is designed around participants meeting public, verifiable requirements.

## Primary Driver

To provide open, replicable safe spaces to learn, to explore, and to empower individuals and organizations so they can solve meaningful problems.

## What Problem This Solves

- builders often work in isolation
- progress is hard to see
- there is little push toward real users

{{event_name}} responds with a low-friction structure that rewards:

- consistent participation
- public progress
- real-world delivery
- real-world outcomes

## Format

- Duration: {{event_duration_weeks}} weeks
- Enrollment period: {{enrollment_start}} to {{enrollment_end}}
- Enrollment closes after {{enrollment_end}}
- One participant may only join one project
- Projects may be solo or team-based
- Every contributor must be named during registration

## Key Principle

Qualify through consistent, verifiable participation, not competition.

## What Participants Should Expect

Participants should expect a format built around:

- one official public repository per project
- visible public progress every week
- a final presentation used for verification
- clear qualification requirements for each reward track

This repository documents:

- the public rules
- the {{season_name}} timeline
- the reward structure
- registration expectations
- verification requirements
- public clarifications and official decisions
