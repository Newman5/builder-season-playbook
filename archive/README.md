# Archive

This folder holds snapshots of past event configurations and summaries.

## Purpose

After each event ends, move the event-specific materials here so they:

- are preserved for reference
- don't clutter the active docs
- can be used by future organizers as concrete examples

## Structure Convention

Organize by season or event name:

```
archive/
  season-1/
    event.yml          ← snapshot of config used
    results.md         ← participation summary and payout totals
    notes.md           ← what worked, what didn't, lessons learned
  season-2/
    ...
```

## What to Archive

- A copy of `config/event.yml` as it was when the event ran
- A results summary (number of participants, qualified builders, payout amounts)
- An honest retrospective (lessons learned, what to change next time)

## Why Archive Matters

This playbook gets better through real use.

The archive is the institutional memory of the project. Future organizers reading past `notes.md` files will learn faster and make fewer mistakes.

Don't skip the retrospective.
